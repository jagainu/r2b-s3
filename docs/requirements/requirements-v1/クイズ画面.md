# 要件定義書: クイズ画面

## 目的

猫の種類を覚えるためのメイン学習画面。写真→種類名・種類名→写真の2方向でランダム出題し、間違えた問題を優先して繰り返すことで効率的な定着を促す。

## 機能

| # | 機能 | 説明 | v1からの変更点 |
|---|------|------|--------------|
| 1 | 写真→種類名クイズ | 猫の写真を見て4択から種類名を選ぶ | 新規 |
| 2 | 種類名→写真クイズ | 猫の種類名を見て4択から写真を選ぶ | 新規 |
| 3 | ランダム出題形式 | 上記2形式をランダムに出題する | 新規 |
| 4 | 進捗表示 | 現在の問題番号（例：3/10）を表示する | 新規 |
| 5 | 優先出題 | 端末に保存された誤答履歴を参照し、間違えた問題を優先して出題する | 新規 |

## Input / Process / Output

| 機能 | Input | Process | Output |
|------|-------|---------|--------|
| 出題（写真→種類名） | セッション開始・問題番号 | 誤答履歴を参照して優先猫を決定。対象猫の写真＋正解1つ＋不正解3つをランダム選択 | 猫の写真＋4つの種類名選択肢を表示 |
| 出題（種類名→写真） | セッション開始・問題番号 | 誤答履歴を参照して優先猫を決定。対象猫の種類名＋正解写真1つ＋不正解写真3つをランダム選択 | 種類名テキスト＋4つの猫写真選択肢を表示 |
| 回答選択 | ユーザーの選択肢タップ | 正解判定。不正解の場合は誤答履歴を端末に保存 | P003（解説画面）へ遷移。正解・不正解フラグを渡す |
| 進捗表示 | 現在の問題番号 | カウンターをインクリメント | 問題番号表示を更新（例：3/10） |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| 問題エリア | 出題 | ローディング中 / 表示中（写真 or テキスト） |
| 選択肢ボタン × 4 | 回答選択 | 通常 / タップ中 / 選択済み（回答後は全ボタン無効化） |
| 問題番号 | 進捗表示 | 回答するたびにカウントアップ（1/10 → 2/10...） |
