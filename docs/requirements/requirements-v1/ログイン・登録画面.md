# 要件定義書: ログイン・登録画面

## 目的

ユーザーがアカウントを作成またはログインする画面。認証後にホーム画面へ遷移する。JWT（httpOnly Cookie）で認証状態を管理する。

## 機能

| # | 機能 | 説明 | v1からの変更点 |
|---|------|------|--------------|
| 1 | メール＋パスワードログイン | メールアドレスとパスワードで認証する | 新規 |
| 2 | Google ログイン | Google OAuth 2.0 で認証する | 新規 |
| 3 | 新規登録 | メールアドレス・パスワード・ユーザー名でアカウントを作成する | 新規 |
| 4 | フォーム切り替え | ログインフォームと新規登録フォームをクライアントサイドで切り替える | 新規 |

## Input / Process / Output

| 機能 | Input | Process | Output |
|------|-------|---------|--------|
| メール＋パスワードログイン | メールアドレス・パスワード | バックエンドで認証。JWT を httpOnly Cookie に発行 | P001（ホーム画面）へ遷移 |
| Google ログイン | Google OAuth 認可コード | Google API で認証。ユーザー情報取得。JWT を httpOnly Cookie に発行 | P001（ホーム画面）へ遷移 |
| 新規登録 | メールアドレス・パスワード・ユーザー名 | メール重複チェック。パスワードをハッシュ化して保存。JWT を httpOnly Cookie に発行 | P001（ホーム画面）へ遷移 |
| フォーム切り替え | リンククリック | クライアントサイドで表示フォームを切り替え（ログイン⇔登録） | フォームUI切り替え |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| メールアドレス入力欄 | ログイン・登録 | 通常 / フォーカス / エラー（赤枠＋メッセージ） |
| パスワード入力欄 | ログイン・登録 | 通常 / フォーカス / エラー |
| ユーザー名入力欄 | 新規登録 | 非表示（ログイン時）/ 通常 / フォーカス / エラー |
| 「ログイン」「登録」ボタン | ログイン・登録 | 通常 / クリック中 / ローディング / エラー時無効化 |
| エラーメッセージ | 全機能 | 非表示 / 表示（認証失敗・バリデーションエラー） |

## バリデーション

| 項目 | ルール |
|------|--------|
| メールアドレス | 必須・メール形式 |
| パスワード | 必須・8文字以上 |
| ユーザー名 | 必須（登録時）・2文字以上・20文字以内 |
