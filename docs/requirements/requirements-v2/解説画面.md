# 要件定義書 v2: 解説画面

## 目的

クイズ回答後に表示される画面。正解・不正解に関わらず猫の特徴と似た種類を提示し、「なぜこの種類か」を理解することで定着を促す。

## 機能

| # | 機能 | 説明 | 対応DBテーブル |
|---|------|------|--------------|
| 1 | 正解・不正解表示 | 回答が正解か不正解かを視覚的に表示する | なし（前画面から正解フラグを受け取る） |
| 2 | 猫の特徴表示 | 正解の猫の写真・種類名・毛色・模様・毛の長さを表示する | `cat_breeds`, `cat_photos`, `coat_colors`, `coat_patterns`, `coat_lengths` READ |
| 3 | 似た種類表示 | 見た目が似ている猫の種類名＋写真を一覧表示する | `similar_cats`, `cat_breeds`, `cat_photos` READ |
| 4 | 次の問題へ | 次の問題（クイズ画面）または10問完了後に結果画面へ遷移する | `quiz_sessions` UPDATE, `session_results` INSERT（finalize API経由・10問完了時のみ） |

## Input / Process / Output

| 機能 | Input | Process | Output | 対応DBテーブル |
|------|-------|---------|--------|--------------|
| 正解・不正解表示 | 直前クイズの正解フラグ（URL パラメータ or 遷移状態） | フラグを評価 | 正解バナー（緑）または不正解バナー（赤）を表示 | なし |
| 猫の特徴表示 | 正解の猫ID | 猫IDをキーに `cat_breeds` JOIN `coat_colors`, `coat_patterns`, `coat_lengths` を取得。`cat_photos` から写真URL取得 | 猫の写真・種類名・特徴リスト（毛色・模様・毛の長さ）を表示 | `cat_breeds` READ, `cat_photos` READ, `coat_colors` READ, `coat_patterns` READ, `coat_lengths` READ |
| 似た種類表示 | 正解の猫ID | `similar_cats` から手動登録の類似猫を取得（priority 降順）。不足分は毛色・模様・毛の長さが1つ以上一致する猫を `cat_breeds` から自動抽出。最大3件を返す | 似た猫の種類名＋写真を最大3件表示 | `similar_cats` READ, `cat_breeds` READ, `cat_photos` READ |
| 次の問題へ | 「次の問題へ」ボタンクリック・現在の問題番号・session_id・認証済みユーザーID | 問題番号が10未満 → 次の問題へ遷移。10問完了 → POST /quiz/sessions/{session_id}/finalize を呼び出す。**サーバーが `quiz_answers` を集計して correct_count/incorrect_count を算出し `session_results` に INSERT**（クライアント申告値は使用しない） | P002（クイズ画面）または P006（結果画面）へ遷移 | `quiz_sessions` UPDATE, `session_results` INSERT（10問完了時のみ） |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| 正解・不正解バナー | 正解・不正解表示 | 正解（緑・「正解！」）/ 不正解（赤・「不正解」） |
| 猫の写真 | 猫の特徴表示 | ローディング中 / 表示中 |
| 特徴リスト（毛色・模様・毛の長さ） | 猫の特徴表示 | ローディング中 / 表示中 |
| 似た種類エリア | 似た種類表示 | ローディング中 / 表示中（最大3件）/ 該当なし（非表示） |
| 「次の問題へ」ボタン | 次の問題へ | 通常 / クリック中。10問完了時はラベルが「結果を見る」に変化 |

## 補足

- 猫の特徴データは `cat_breeds` + `coat_*` マスターを JOIN して取得（1回のAPIコール）
- 似た種類の自動抽出ロジック：`similar_cats.priority = 0` に相当する猫を `cat_breeds` から取得（毛色 OR 模様 OR 毛の長さが一致）
- 10問完了時は `POST /quiz/sessions/{session_id}/finalize` を呼び出し、サーバーが `quiz_answers` を集計して `session_results` に INSERT。完了後 P006（結果画面）へリダイレクト
