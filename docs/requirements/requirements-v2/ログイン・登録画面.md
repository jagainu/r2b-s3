# 要件定義書 v2: ログイン・登録画面

## 目的

ユーザーがアカウントを作成またはログインする画面。認証後にホーム画面へ遷移する。JWT（httpOnly Cookie）で認証状態を管理する。

## 機能

| # | 機能 | 説明 | 対応DBテーブル |
|---|------|------|--------------|
| 1 | メール＋パスワードログイン | メールアドレスとパスワードで認証する | `users` READ |
| 2 | Google ログイン | Google OAuth 2.0 で認証する | `users` READ / WRITE |
| 3 | 新規登録 | メールアドレス・パスワード・ユーザー名でアカウントを作成する | `users` WRITE |
| 4 | フォーム切り替え | ログインフォームと新規登録フォームをクライアントサイドで切り替える | なし |

## Input / Process / Output

| 機能 | Input | Process | Output | 対応DBテーブル |
|------|-------|---------|--------|--------------|
| メール＋パスワードログイン | メールアドレス・パスワード | `users` テーブルでメール照合・bcrypt パスワード検証。JWT を httpOnly Cookie に発行 | P001（ホーム画面）へ遷移 | `users` READ |
| Google ログイン | Google OAuth 認可コード | Google API でユーザー情報取得。`users.google_id` で照合。未登録なら新規 INSERT。JWT を httpOnly Cookie に発行 | P001（ホーム画面）へ遷移 | `users` READ / INSERT |
| 新規登録 | メールアドレス・パスワード・ユーザー名 | `users.email` の重複チェック。パスワードを bcrypt でハッシュ化して `users` に INSERT。JWT を httpOnly Cookie に発行 | P001（ホーム画面）へ遷移 | `users` READ / INSERT |
| フォーム切り替え | リンククリック | クライアントサイドで表示フォームを切り替え（ログイン⇔登録） | フォームUI切り替え | なし |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| メールアドレス入力欄 | ログイン・登録 | 通常 / フォーカス / エラー（赤枠＋メッセージ） |
| パスワード入力欄 | ログイン・登録 | 通常 / フォーカス / エラー |
| ユーザー名入力欄 | 新規登録 | 非表示（ログイン時）/ 通常 / フォーカス / エラー |
| 「ログイン」「登録」ボタン | ログイン・登録 | 通常 / クリック中 / ローディング（API 呼び出し中）/ エラー時無効化 |
| エラーメッセージ | 全機能 | 非表示 / 表示（認証失敗・メール重複・バリデーションエラー） |

## バリデーション

| 項目 | ルール |
|------|--------|
| メールアドレス | 必須・メール形式 |
| パスワード | 必須・8文字以上 |
| ユーザー名 | 必須（登録時）・2文字以上・20文字以内 |

## 補足

- 未認証時に認証必須ページへアクセスした場合、本画面へリダイレクト
- Google ログイン成功時、`users.google_id` がすでに存在する場合はログイン処理（INSERT なし）
