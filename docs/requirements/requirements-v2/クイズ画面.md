# 要件定義書 v2: クイズ画面

## 目的

猫の種類を覚えるためのメイン学習画面。写真→種類名・種類名→写真の2方向でランダム出題し、間違えた問題を優先して繰り返すことで効率的な定着を促す。

## 機能

| # | 機能 | 説明 | 対応DBテーブル |
|---|------|------|--------------|
| 1 | 写真→種類名クイズ | 猫の写真を見て4択から種類名を選ぶ | `cat_breeds`, `cat_photos`, `wrong_answers` READ |
| 2 | 種類名→写真クイズ | 猫の種類名を見て4択から写真を選ぶ | `cat_breeds`, `cat_photos`, `wrong_answers` READ |
| 3 | ランダム出題形式 | 上記2形式をランダムに出題する | - |
| 4 | 進捗表示 | 現在の問題番号（例：3/10）を表示する | なし |
| 5 | 優先出題 | DBの誤答履歴（`wrong_answers`）を参照し、間違えた問題を優先して出題する | `wrong_answers` READ |
| 6 | 回答保存 | 回答結果をDBに保存する | `wrong_answers` UPSERT, `correct_answers` INSERT |

## Input / Process / Output

| 機能 | Input | Process | Output | 対応DBテーブル |
|------|-------|---------|--------|--------------|
| 出題（写真→種類名） | セッション開始・問題番号・認証済みユーザーID | `wrong_answers` からユーザーの誤答履歴を取得。wrong_count を重みとして優先猫を決定。対象猫の写真（display_order=1）＋正解種類名1つ＋不正解種類名3つをランダム選択 | 猫の写真＋4つの種類名選択肢を表示 | `wrong_answers` READ, `cat_breeds` READ, `cat_photos` READ |
| 出題（種類名→写真） | セッション開始・問題番号・認証済みユーザーID | `wrong_answers` からユーザーの誤答履歴を取得。wrong_count を重みとして優先猫を決定。対象猫の種類名＋正解写真（display_order=1）1つ＋不正解写真3つをランダム選択 | 種類名テキスト＋4つの猫写真選択肢を表示 | `wrong_answers` READ, `cat_breeds` READ, `cat_photos` READ |
| 回答選択 | ユーザーの選択肢クリック・認証済みユーザーID | 正解判定。不正解の場合は `wrong_answers` に UPSERT（wrong_count += 1, last_wrong_at 更新）。正解の場合は `correct_answers` に INSERT（重複なし） | P003（解説画面）へ遷移。正解フラグ＋猫IDを渡す | `wrong_answers` UPSERT, `correct_answers` INSERT |
| 進捗表示 | 現在の問題番号 | カウンターをインクリメント | 問題番号表示を更新（例：3/10） | なし |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| 問題エリア | 出題 | ローディング中 / 表示中（写真 or テキスト） |
| 選択肢ボタン × 4 | 回答選択 | 通常 / クリック中 / 選択済み（回答後は全ボタン無効化） |
| 問題番号 | 進捗表示 | 回答するたびにカウントアップ（1/10 → 2/10...） |

## 補足

- セッション内（10問）は同じ猫が重複して出題されないようサーバーサイドで管理
- 誤答履歴がない場合（初回ユーザー等）はランダム出題にフォールバック
- `wrong_answers` の UPSERT は `ON CONFLICT (user_id, cat_breed_id) DO UPDATE`
