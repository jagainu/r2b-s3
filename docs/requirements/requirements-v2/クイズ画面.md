# 要件定義書 v2: クイズ画面

## 目的

猫の種類を覚えるためのメイン学習画面。写真→種類名・種類名→写真の2方向でランダム出題し、間違えた問題を優先して繰り返すことで効率的な定着を促す。

## 機能

| # | 機能 | 説明 | 対応DBテーブル |
|---|------|------|--------------|
| 1 | 写真→種類名クイズ | 猫の写真を見て4択から種類名を選ぶ | `quiz_sessions`, `quiz_questions`, `quiz_choices`, `cat_breeds`, `cat_photos` READ |
| 2 | 種類名→写真クイズ | 猫の種類名を見て4択から写真を選ぶ | `quiz_sessions`, `quiz_questions`, `quiz_choices`, `cat_breeds`, `cat_photos` READ |
| 3 | ランダム出題形式 | 上記2形式をランダムに出題する | - |
| 4 | 進捗表示 | 現在の問題番号（例：3/10）を表示する | なし |
| 5 | 優先出題 | DBの誤答履歴（`wrong_answers`）を参照し、間違えた問題を優先して出題する | `wrong_answers` READ |
| 6 | 回答保存 | 回答結果をDBに保存する | `quiz_answers` INSERT, `wrong_answers` UPSERT, `correct_answers` INSERT |

## Input / Process / Output

| 機能 | Input | Process | Output | 対応DBテーブル |
|------|-------|---------|--------|--------------|
| 出題（写真→種類名） | セッション開始・認証済みユーザーID | POST /quiz/sessions を呼び出す。`wrong_answers` を参照し優先猫を決定。全10問分の `quiz_sessions`・`quiz_questions`・`quiz_choices` をDBに保存。**正解（correct_cat_breed_id）はDBのみ保持しレスポンスに含めない** | 猫の写真＋4つの種類名選択肢を表示（10問分一括） | `wrong_answers` READ, `quiz_sessions` INSERT, `quiz_questions` INSERT, `quiz_choices` INSERT, `cat_breeds` READ, `cat_photos` READ |
| 出題（種類名→写真） | セッション開始・認証済みユーザーID | POST /quiz/sessions 内でランダムに種類名→写真形式を割り当て。quiz_choices に photo_url を保存 | 種類名テキスト＋4つの猫写真選択肢を表示 | `quiz_sessions` INSERT, `quiz_questions` INSERT, `quiz_choices` INSERT |
| 回答選択 | ユーザーの選択肢クリック・session_id・question_number・selected_cat_id | POST /quiz/answer を呼び出す。**正解判定はサーバーが `quiz_questions.correct_cat_breed_id` と照合**（クライアント申告値に依存しない）。UNIQUE(session_id, question_number)で二重回答防止。quiz_answersにINSERT。不正解→wrong_answers UPSERT、正解→correct_answers INSERT | P003（解説画面）へ遷移。正解フラグ＋猫IDを渡す | `quiz_answers` INSERT, `wrong_answers` UPSERT, `correct_answers` INSERT |
| 進捗表示 | 現在の問題番号 | カウンターをインクリメント | 問題番号表示を更新（例：3/10） | なし |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| 問題エリア | 出題 | ローディング中 / 表示中（写真 or テキスト） |
| 選択肢ボタン × 4 | 回答選択 | 通常 / クリック中 / 選択済み（回答後は全ボタン無効化） |
| 問題番号 | 進捗表示 | 回答するたびにカウントアップ（1/10 → 2/10...） |

## 補足

- セッション内（10問）は同じ猫が重複して出題されないようサーバーサイドで管理
- 誤答履歴がない場合（初回ユーザー等）はランダム出題にフォールバック
- `wrong_answers` の UPSERT は `ON CONFLICT (user_id, cat_breed_id) DO UPDATE`
- **正解情報（correct_cat_breed_id）はサーバーのDBのみ保持し、APIレスポンスには含めない**（改ざん防止）
- `quiz_answers` の `UNIQUE(session_id, question_number)` により同一問題への二重回答を防止
