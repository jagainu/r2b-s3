# 要件定義書 v2: ホーム画面

## 目的

アプリのエントリーポイント。クイズを開始する起点となる画面。今日の一匹をクイズ形式で提示し、毎日の学習習慣を促す。

## 機能

| # | 機能 | 説明 | 対応DBテーブル |
|---|------|------|--------------|
| 1 | クイズ開始 | クイズ画面（P002）へ遷移してクイズを開始する | なし |
| 2 | 今日の一匹 | 認証済みユーザーごとに1日1回変わるランダムな猫をクイズ形式で出題する | `cat_breeds`, `cat_photos` READ |
| 3 | 今日の一匹（回答） | 選択肢クリックで正解判定し、解説画面へ遷移する | `wrong_answers` UPSERT / `correct_answers` INSERT |

## Input / Process / Output

| 機能 | Input | Process | Output | 対応DBテーブル |
|------|-------|---------|--------|--------------|
| クイズ開始 | 「クイズを始める」ボタンクリック・認証済みユーザーID | セッション状態を初期化 | P002（クイズ画面）へ遷移 | なし |
| 今日の一匹 | 現在日付・認証済みユーザーID | 日付＋ユーザーIDをシードにランダム猫を1種選択。同日中は同じ猫を返す。`cat_breeds` から4択選択肢を生成 | 猫の写真＋4択クイズを表示 | `cat_breeds` READ, `cat_photos` READ |
| 今日の一匹（回答） | ユーザーの選択肢クリック・session_id・question_number・selected_cat_id | POST /quiz/answer を呼び出す。**正解判定はサーバーが quiz_questions.correct_cat_breed_id と照合**（クライアント申告値不使用）。不正解→ `wrong_answers` UPSERT、正解→ `correct_answers` INSERT | P003（解説画面）へ遷移 | `quiz_answers` INSERT, `wrong_answers` UPSERT, `correct_answers` INSERT |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| 「クイズを始める」ボタン | クイズ開始 | 通常 / クリック中（押下エフェクト） |
| 今日の一匹エリア | 今日の一匹 | ローディング中 / 未回答（クイズ表示） / 回答済み（正解・不正解表示） |
| 選択肢ボタン × 4 | 今日の一匹（回答） | 通常 / クリック中 / 正解（緑） / 不正解（赤）/ 回答後は全ボタン無効化 |

## 補足

- 今日の一匹は認証済みユーザー向け機能（ユーザーIDをシードに使用）
- 今日の一匹の出題猫は、同一ユーザー・同一日付で常に同一猫を返す（サーバーサイドで日付+ユーザーIDをシードとして計算）
