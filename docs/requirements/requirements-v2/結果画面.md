# 要件定義書 v2: 結果画面

## 目的

10問のクイズ完了後に表示される画面。正答率・覚えた種類数を表示し、継続学習を促す。

## 機能

| # | 機能 | 説明 | 対応DBテーブル |
|---|------|------|--------------|
| 1 | 結果表示 | 今回セッションの正答率・正解数・不正解数を表示する | `session_results` READ（直近1件） |
| 2 | 累計表示 | DBに保存された累計の覚えた種類数（ユニーク正解猫数）を表示する | `correct_answers` READ（COUNT） |
| 3 | もう一度挑戦 | ホーム画面（P001）へ戻る | なし |

## Input / Process / Output

| 機能 | Input | Process | Output | 対応DBテーブル |
|------|-------|---------|--------|--------------|
| 結果表示 | 認証済みユーザーID | `GET /users/me/sessions/latest?source=quiz` を呼び出す（source=quiz 固定）。サーバー算出済みの正解数・不正解数を取得。正答率はクライアントで計算（正解数 / (正解数 + 不正解数)） | 正答率・正解数・不正解数を表示 | `session_results` READ（source='quiz', user_id で最新1件） |
| 累計表示 | 認証済みユーザーID | `correct_answers` で `user_id = ?` の件数を COUNT（UNIQUE制約により自動でユニーク数） | 累計覚えた種類数を表示 | `correct_answers` READ |
| もう一度挑戦 | 「もう一度挑戦」ボタンクリック | - | P001（ホーム画面）へ遷移 | なし |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| 正答率 | 結果表示 | ローディング中 / 表示中 |
| 正解数・不正解数 | 結果表示 | ローディング中 / 表示中 |
| 累計覚えた種類数 | 累計表示 | ローディング中 / 表示中 |
| 「もう一度挑戦」ボタン | もう一度挑戦 | 通常 / クリック中 |

## 補足

- 結果表示に必要なデータは解説画面の「次の問題へ（10問完了時）」処理（`POST /quiz/sessions/{id}/finalize`）で `session_results` に INSERT済み
- `session_results` の `source='quiz'` かつ `user_id` で最新1件を取得（`source` で今日の一匹と区別）
- `GET /users/me/sessions/latest?source=quiz` の `source=quiz` パラメータは固定（ハードコード）
- 累計覚えた種類数は `COUNT(*)` on `correct_answers WHERE user_id = ?`（UNIQUE制約により既にユニーク集計）
- セッション結果とユーザー統計を1回のAPIコールでまとめて取得してもよい
