## エンティティ関係サマリー

```
users
  ├── wrong_answers (1:N)
  ├── correct_answers (1:N)
  ├── quiz_sessions (1:N)
  └── session_results (1:N)

cat_breeds
  ├── coat_colors (N:1)
  ├── coat_patterns (N:1)
  ├── coat_lengths (N:1)
  ├── cat_photos (1:N)
  ├── similar_cats (1:N 双方向)
  ├── wrong_answers (1:N)
  ├── correct_answers (1:N)
  └── quiz_questions (1:N, correct_cat_breed_id)

quiz_sessions
  ├── quiz_questions (1:N, UNIQUE: session_id + question_number)
  │     └── quiz_choices (1:N, FK: question_id)
  └── quiz_answers (1:N, FK: session_id + question_number, UNIQUE)

session_results
  └── quiz_sessions (N:1, session_id UNIQUE)
```

## 補足

- 猫データ（cat_breeds・cat_photos・マスター）はシードデータとして初期投入
- 写真はS3に保存し、URLをDBに持つ
- 類似猫の自動抽出ロジック：毛色・模様・毛の長さが1つ以上一致する猫（手動登録を優先）
